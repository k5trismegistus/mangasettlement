# 要件定義ドラフト v0.1

## 目的

Linux PC 内の `library/` に置いた zip 画像ライブラリを、家庭内LAN上の iPhone のブラウザから快適に閲覧できるWebアプリを作る。

---

## 想定環境

- サーバー: Linux PC
- クライアント: iPhone Air のWebブラウザ
- 利用範囲: 家庭内LANのみ
- 利用者: 自分のみ
- 認証: Basic認証

### 起動方法

- npm install
- npm run init
- npm run start

スマホから `http://PCのIP:ポート` でアクセスする。

---

## 技術構成

- Node.js
- TypeScript
- Fastify
- React
- Vite
- SQLite
- zip内画像は展開保存せず、表示時にzipから直接読み込む
- Web画面とAPIは同一Nodeサーバーで提供する

## データインポート

- `library/` にzipファイルを配置する
- トップページの「リフレッシュ」でスキャン開始
- zipファイル本体には一切変更を加えない
- zipの識別は `sha256`
- リフレッシュ時、ファイルサイズ・更新日時が同じものは再解析しない
- 変化があるzipのみsha256を再計算する
- ファイル名変更には追従する
- 削除されたzipはリフレッシュ時に一覧から除外する
- zip内フォルダ階層に対応する
- 画像以外のファイルは無視する
- 日本語ファイル名zipの文字化けに対応する
- ページ順は自然順ソートとする

### ソート例

- 1.jpg
- 2.jpg
- 10.jpg

---

## 対応画像形式

ブラウザ表示可能な形式を対象とする。

- jpg
- jpeg
- png
- webp
- gif
- avif（ブラウザ依存）

---

## サムネイル

- 全ページ分のサムネイルを生成する
- サイズは300px四方程度
- 保存先: `thumbnail/<zip_sha256>/<page_key>.webp`
- リフレッシュ後にバックグラウンド生成する
- 未生成ページはプレースホルダー画像を表示する
- 生成失敗ページはエラー画像を表示する
- 失敗してもページ自体は存在扱いとする

---

## トップページ

### 表示内容

- 表紙サムネイル
- ファイル名
- ページ数
- タグ
- 存在しないzipの警告表示

### 機能

- ライブラリ一覧表示
- ファイル名検索
- タグ検索
- 更新日時順ソート
- リフレッシュボタン

---

## 個別ライブラリページ

### 機能

- zip内画像をコミックビューア形式で表示する
- 1ページ表示 / 2ページ見開きを切り替え可能
- 右綴じをデフォルトとする
- 左綴じ / 右綴じ切り替え可能
- 表紙は単ページ表示
- 見開き開始位置を1ページずらせる
- ページサムネイル一覧表示
- サムネイルからページジャンプ可能
- スマホ向けタップ・スワイプ操作対応
- 基本は画面幅フィット
- iPhoneの2本指ズームを妨げない
- 現在ページの前後2ページを先読みする

### MVP対象外

- 最後に読んだページの記録

---

## タグ・メモ

- zip単位でタグを付与できる
- zip単位でメモを付与できる
- 検索対象はファイル名 + タグ
- メモ全文検索はMVP対象外

---

## API案

- GET `/api/libraries`
- POST `/api/refresh`
- GET `/api/libraries/:id`
- GET `/api/libraries/:id/pages`
- GET `/api/libraries/:id/pages/:page/image`
- GET `/api/libraries/:id/pages/:page/thumbnail`
- PUT `/api/libraries/:id/tags`
- PUT `/api/libraries/:id/memo`

---

## 非機能要件

- 想定zip数: 1000個
- 想定zipサイズ: 1ファイル最大500MB程度
- 同時アクセス: 最大2台程度
- 重視点: 閲覧時の快適さ
- バックアップ要件: なし
- ポータブル性: SQLite + `library/` + `thumbnail/` + `.env` で移行可能

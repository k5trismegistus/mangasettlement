# エラー設計ドラフト v0.1

## 方針

- エラーはAPI・UI・ジョブで一貫した扱いにする
- zipファイル本体には一切変更を加えない
- 一部ページの失敗でライブラリ全体を利用不能にしない
- ユーザーが原因を把握できる短いメッセージを表示する
- 詳細ログはDBまたはサーバーログに残す

---

# エラー分類

1. 認証エラー
2. 入力エラー
3. ライブラリ欠損エラー
4. ページ未存在エラー
5. zip読み込みエラー
6. 画像読み込みエラー
7. サムネイル生成エラー
8. ジョブ多重起動エラー
9. 内部エラー

---

# APIエラー形式

## 基本形式

- `error`
  - `code`
  - `message`

## 例

- `error.code`: `PAGE_NOT_FOUND`
- `error.message`: `Page not found`

---

# エラーコード一覧

## AUTH_REQUIRED

### 発生条件

- Basic認証に失敗した
- 認証情報が未指定または不正

### HTTP Status

- 401 Unauthorized

### UI表示

- ブラウザ標準のBasic認証ダイアログを表示する

---

## INVALID_REQUEST

### 発生条件

- APIリクエストの形式が不正
- 必須パラメータがない
- `limit` や `offset` が不正
- タグ更新リクエストが配列ではない

### HTTP Status

- 400 Bad Request

### UI表示

- `リクエストが正しくありません`

---

## NOT_FOUND

### 発生条件

- 指定されたAPIリソースが存在しない
- 存在しないlibrary idを指定した
- 存在しないjob idを指定した

### HTTP Status

- 404 Not Found

### UI表示

- `対象が見つかりません`

---

## LIBRARY_MISSING

### 発生条件

- DB上には存在するが、zipファイル本体が `library/` に存在しない
- `is_missing = 1` のライブラリに画像取得リクエストを行った

### HTTP Status

- 404 Not Found

### UI表示

- `zipファイルが見つかりません`

---

## PAGE_NOT_FOUND

### 発生条件

- 指定されたページ番号が存在しない
- ページ番号が範囲外

### HTTP Status

- 404 Not Found

### UI表示

- `ページが見つかりません`

---

## ZIP_READ_ERROR

### 発生条件

- zipファイルを開けない
- zipファイルが壊れている
- zip内エントリを読み込めない
- zipファイルの文字コード解釈に失敗した

### HTTP Status

- 500 Internal Server Error

### UI表示

- `zipファイルを読み込めません`

---

## IMAGE_READ_ERROR

### 発生条件

- zip内画像を読み込めない
- 画像形式が壊れている
- ブラウザ表示対象として扱えない

### HTTP Status

- 500 Internal Server Error

### UI表示

- `画像を表示できません`

---

## THUMBNAIL_ERROR

### 発生条件

- サムネイル生成に失敗した
- サムネイルファイルの保存に失敗した
- 画像変換処理に失敗した

### HTTP Status

- サムネイル取得APIでは200を返し、エラー画像を返す
- ジョブ状態ではページ単位で `thumb_status = error`

### UI表示

- エラー画像を表示する

---

## REFRESH_ALREADY_RUNNING

### 発生条件

- refreshジョブ実行中に再度refreshを開始しようとした

### HTTP Status

- 409 Conflict

### UI表示

- `更新処理はすでに実行中です`

---

## INTERNAL_ERROR

### 発生条件

- 想定外のサーバーエラー
- DBエラー
- ファイルシステムエラー
- その他未分類のエラー

### HTTP Status

- 500 Internal Server Error

### UI表示

- `内部エラーが発生しました`

---

# UI別エラー表示

## トップページ

### リフレッシュ失敗

表示:

- `更新に失敗しました`

詳細:

- 可能であればジョブメッセージを表示する

### ライブラリ欠損

表示:

- カード上に `ファイルなし` と表示する
- 通常一覧では欠損ライブラリは非表示

### サムネイル未生成

表示:

- プレースホルダー画像

### サムネイル生成失敗

表示:

- エラー画像

---

## 個別ライブラリページ

### ライブラリ欠損

表示:

- `zipファイルが見つかりません`
- トップへ戻るボタンを表示する

### ページ未存在

表示:

- `ページが見つかりません`
- 先頭ページへ戻るボタンを表示する

### 画像読み込み失敗

表示:

- ビューア領域にエラー画像またはエラーメッセージを表示する
- ページ送りは継続可能にする

### サムネイル読み込み失敗

表示:

- ページサムネイル一覧にエラー画像を表示する

---

# ジョブ別エラー処理

## refreshジョブ

### zip単位の失敗

- 対象zipの登録または更新を失敗扱いにする
- 他のzipの処理は継続する
- job messageに失敗件数を記録する
- 全体として処理が完了した場合、job statusはdoneにする
- すべての処理が失敗した場合のみjob statusをerrorにする

---

## thumbnail_generateジョブ

### ページ単位の失敗

- 対象ページの `thumb_status = error`
- 他ページのサムネイル生成は継続する
- ジョブ全体はdoneにする

### ライブラリ全体が読めない場合

- job statusをerrorにする
- messageに理由を保存する

---

# ログ方針

## サーバーログ

記録する内容:

- エラー発生時刻
- エラーコード
- 対象library id
- 対象page no
- 対象file path
- 例外メッセージ

## DB上のジョブログ

記録する内容:

- job status
- progress
- message
- created_at
- started_at
- finished_at

# エラー設計の決定事項

- APIエラーは `{ error: { code, message } }` 形式に統一する
- サムネイル取得APIは未生成・生成失敗時も画像を返す
- 一部zipの失敗でrefresh全体は止めない
- 一部ページのサムネイル生成失敗でジョブ全体は止めない
- 欠損zipは通常一覧では非表示
- 画像表示失敗時もページ送りは継続可能にする
- 詳細エラーはUIに出しすぎず、ログに残す

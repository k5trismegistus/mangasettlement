# 実装フェーズ分解ドラフト v0.1

## 方針

- 先に「使える最小構成」を作る
- その後に快適性・管理機能・安定性を追加する
- 各フェーズごとに動作確認可能な状態で区切る
- iPhone実機確認を各フェーズで行う

---

# 全体ロードマップ

1. Phase 0: プロジェクト基盤構築
2. Phase 1: 最小閲覧機能
3. Phase 2: サムネイルと一覧UI改善
4. Phase 3: タグ・メモ・検索
5. Phase 4: コミックビューア強化
6. Phase 5: ジョブ管理と安定化
7. Phase 6: 品質改善・運用準備

---

# Phase 0: プロジェクト基盤構築

## 目的

開発開始できる土台を作る。

## 実装内容

- Node.js + TypeScript 初期化
- Fastify導入
- React + Vite構築
- ESLint / Prettier 設定
- tsconfig整備
- ディレクトリ作成
- `.env.example` 作成
- Basic認証ミドルウェア実装
- SQLite接続処理
- マイグレーション仕組み作成
- `npm run init`
- `npm run start`

## 完了条件

- サーバー起動できる
- Basic認証がかかる
- DB初期化できる
- React画面が表示される

# Phase 1: 最小閲覧機能

## 目的

zipを読み込み、1ページ表示で閲覧できる状態にする。

## 実装内容

### バックエンド

- `library/` スキャン処理
- zip一覧取得API
- ライブラリ詳細API
- ページ一覧API
- 原寸画像API
- refresh API（同期簡易版でも可）

### フロントエンド

- トップページ作成
- ライブラリ一覧表示
- 個別ライブラリページ作成
- 1ページビューア
- ページ送り
- トップへ戻る

## 完了条件

- zipを置いてリフレッシュできる
- 一覧から開ける
- 画像が読める
- iPhoneで閲覧できる
